────────────────────────────────────────────────────────────────────────────
 COBOL PROGRAMS
────────────────────────────────────────────────────────────────────────────

  payroll_processing.cob (PAYPRO01)
  ─────────────────────────────────
  The core payroll engine. Demonstrates:
  - EXEC SQL embedded in COBOL (cursor, fetch, insert, commit)
  - Decimal COMP-3 arithmetic for tax calculations
  - PERFORM/EVALUATE for structured control flow
  - FD/RECORD WORKING-STORAGE FILE sections
  - REDEFINES for tax table overlay
  - Multi-step journal entry generation
  - Printed register with formatted numeric output (PIC Z,ZZ9.99)

  Key logic:
  1. Open cursor on EMPLOYEE WHERE ACTIVE_FLAG = 'Y'
  2. For hourly employees, SUM hours from TIMECARD (SQL aggregate)
  3. Calculate gross pay (salary/26 or hourly * hours + OT * 1.5)
  4. Apply Federal withholding tax (tiered bracket method)
  5. Apply Illinois state tax (4.95% flat, less exemptions)
  6. Compute FICA: Social Security (6.2%) + Medicare (1.45%)
  7. Apply benefit deductions: Health ($125), Dental ($18.50), 401k (4%)
  8. Net pay = Gross - All deductions
  9. INSERT into PAYCHECK table
  10. Write GL journal: Dr Wages Expense (6100), Cr Payroll Cash (1020)
      Cr Fed Tax Withheld (2300), Cr State Tax (2310), Cr FICA (2320)
  11. Print formatted payroll register

 CICS ONLINE TRANSACTIONS
────────────────────────────────────────────────────────────────────────────

  customer_inquiry.cics (CINQPRO1, Transaction: CINQ)
  ────────────────────────────────────────────────────
  Customer service terminal inquiry screen. Demonstrates:
  - EXEC CICS RECEIVE MAP / SEND MAP (BMS mapset processing)
  - EXEC CICS ASSIGN AID (PF key detection)
  - EXEC CICS XCTL (transfer control to linked program)
  - EXEC CICS RETURN TRANSID COMMAREA (pseudo-conversational design)
  - SQL cursor for open orders (last 5 FETCH FIRST 5 ROWS ONLY)
  - SQL aggregate for AR aging buckets (CASE WHEN in SELECT)
  - EVALUATE WS-AID for PF3/PF5/ENTER branching

────────────────────────────────────────────────────────────────────────────
 JCL JOB STREAMS
────────────────────────────────────────────────────────────────────────────

  payroll_run.jcl (PAYRJOB)
  ─────────────────────────
  Scheduled bi-weekly Friday payroll job stream:
  Step 1: HOUSEKEEPING — IEFBR14 deletes prior work files
  Step 2: TIMVLD01 — Validate timecard hours and codes
  Step 3: IEBGENER — Print timecard error report to INTRDR
  Step 4: PAYPRO01 — Main payroll calculation (COBOL above)
  Step 5: Conditional abort if PAYCALC RC > 4
  Step 6: IEBGENER — Print payroll register from PRNTFILE
  Step 7: SQRRUN   — Generate SQR payroll register report
  Step 8: GLPOST01 — Post GL journal entries (set POSTED_FLAG='Y')
  Step 9: IEBCOPY  — Archive payroll datasets to tape
  Final:  Write completion trigger, notify operator

  daily_batch.jcl (DLYBTCH)
  ─────────────────────────
  Scheduled nightly (01:00 AM) daily operations:
  Step 01: HOUSEKEEPING
  Step 02: ORDFILL1 — Fulfill open orders from inventory
  Step 03: INVPST01 — Post pending inventory transactions
  Step 04: REORDPT1 — Generate reorder alert report
  Step 05: ARAGING1 — Compute AR aging buckets
  Step 06: GLSUM01  — Summarize daily GL activity
  Step 07: SQRRUN   — Three daily management reports
  Step 08: MAILDIST — Email reports to managers
  Step 09: AUDTLOG1 — Record successful completion
  Step 10: Alert operator on any step failure (RC > 4)

  Demonstrates: COND parameters, IF/THEN/ENDIF, cataloged datasets,
  DCB parameters, INTRDR output, conditional notification

 HOW TO USE THESE EXAMPLES IN TIME WARP STUDIO
────────────────────────────────────────────────────────────────────────────

1. SQL SCHEMA        — Load erp_schema.sql to create tables and seed data
                       Run in the SQL language mode

2. COBOL PAYROLL     — Open payroll_processing.cob in COBOL mode
                       Study the DATA DIVISION structure and PROCEDURE DIVISION
                       The EXEC SQL statements show DB2/COBOL interface

3. CICS CUSTOMER INQ — Open customer_inquiry.cics in CICS mode
                       Trace the pseudo-conversational flow:
                       RETURN → AID check → SQL lookup → SEND MAP

4. JCL PAYROLL JOB   — Open payroll_run.jcl in JCL mode
                       Study DD statement structure, COND parameters
                       See how jobs chain: validate → calculate → post → report

5. JCL DAILY BATCH   — Open daily_batch.jcl in JCL mode
                       Observe IF/THEN/ENDIF conditional step execution

6. SQR REPORT        — Open payroll_report.sqr in SQR mode
                       See how begin-SELECT directly queries the database
                       Study break logic for department subtotals

────────────────────────────────────────────────────────────────────────────
 GLOSSARY — MAINFRAME TERMS
────────────────────────────────────────────────────────────────────────────

  BMS       Basic Mapping Support — CICS 3270 screen definition language
  CICS      Customer Information Control System — IBM transaction server
  COMP-3    Packed decimal storage format (2 digits per byte)
  DD        Data Definition — JCL statement defining a dataset
  DSN       Data Set Name — mainframe file identifier
  JCL       Job Control Language — batch job script language
  JES2/JES3 Job Entry Subsystem — z/OS batch job scheduler
  LRECL     Logical Record Length — fixed/variable record size
  OPC/TWS   Operations Planning & Control / Tivoli Workload Scheduler
  PIC       PICTURE clause — COBOL data type definition
  RECFM     Record Format: FB=Fixed/Blocked, FBA=Fixed/Blocked/ANSI
  SQLCA     SQL Communication Area — COBOL/SQL error return structure
  SQR       Structured Query Report Language (PeopleSoft/Oracle)
  SYSIN     Standard input to a program step
  SYSOUT    Directed to JES spool for printer/viewing
  VSAM      Virtual Storage Access Method — indexed mainframe files
